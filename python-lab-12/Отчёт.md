### Ответы на вопросы

1. **Преимущества Flow и StateFlow:** `Flow` реализует реактивное программирование — UI обновляется автоматически при любом изменении данных в Room. `StateFlow` сохраняет последнее состояние и немедленно эмитирует его новым подписчикам, что идеально для привязки к Compose.
2. **ViewModel и повороты экрана:** `ViewModel` хранится в специальном `ViewModelStore`, привязанном к `Activity`, а не её жизненному циклу. При пересоздании Activity (например, при повороте) `ViewModel` *не уничтожается*, что сохраняет загруженные данные и не вызывает повторного обращения к БД.
3. **Room vs прямой SQLite:** Room проверяет корректность SQL-запросов *на этапе компиляции*, избавляет от шаблонного кода (`Cursor`, `ContentValues`) и обеспечивает нативную интеграцию с Coroutines и Flow.

export const addNote = (note: GeoNote): Promise<void> => {
    return new Promise((resolve, reject) => {
        db.transaction(tx => {
            tx.executeSql(
                'INSERT INTO notes (id, title, content, latitude, longitude, address, photoUri, createdAt) VALUES (?, ?, ?, ?, ?, ?, ?, ?)',
                [note.id, note.title, note.content, note.latitude, note.longitude,
                 note.address || null, note.photoUri || null, note.createdAt],
                () => resolve(),
                (_, error) => { reject(error); return false; }
            );
        });
    });
};

### Ответы на вопросы

1. **React Native vs Flutter:** React Native использует нативные UIKit/Android View через «мост» (Bridge/JSI), что обеспечивает *нативный look&feel*. Flutter отрисовывает весь UI самостоятельно через движок Skia/Impeller — и выглядит *одинаково* на всех платформах.
2. **Роль Redux в мобильном приложении:** Redux централизует состояние, доступное на разных экранах (например, список заметок). Это устраняет *prop drilling* и упрощает отладку через Redux DevTools.

## Общий вывод {#вывод}

В ходе лабораторной работы №12 были изучены и практически освоены современные подходы к разработке мобильных приложений:

- **Jetpack Compose** — лучший выбор для чисто Android-приложений, обеспечивает максимальную интеграцию с экосистемой Google и нативную производительность.
- **React Native** — наиболее популярная кроссплатформенная технология благодаря огромной экосистеме npm и знакомому для веб-разработчиков синтаксису React.

Каждая технология имеет свою нишу, и выбор зависит от задачи: качество UI, скорость разработки, нужные платформы и имеющаяся экспертиза команды.
