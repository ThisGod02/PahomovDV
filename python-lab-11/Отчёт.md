- Реализована статистика библиотеки (`/stats`).

### Ответы на вопросы (Часть 1)
1. **В чем преимущества использования Pydantic моделей для валидации?**
   Pydantic обеспечивает строгую типизацию данных в рантайме, автоматически преобразует типы (например, строку в дату) и генерирует подробные сообщения об ошибках при некорректном вводе. Также он интегрируется со Swagger, создавая схемы данных.

2. **Как работает автоматическая документация в FastAPI?**
   FastAPI анализирует аннотации типов Python и модели Pydantic, после чего генерирует спецификацию OpenAPI (JSON). Swagger UI и ReDoc используют эту спецификацию для создания интерактивного веб-интерфейса.

3. **Почему важно проверять уникальность ISBN?**
   ISBN — это уникальный международный стандартный книжный номер. Дублирование ISBN может привести к путанице в каталоге и ошибкам при поиске конкретной книги.

4. **Какие статус-коды HTTP вы использовали и почему?**
   - `200 OK`: Успешные GET, PUT, PATCH запросы.
   - `201 Created`: Успешное создание книги (POST).
   - `204 No Content`: Успешное удаление (DELETE).
   - `400 Bad Request`: Ошибки логики (например, книга уже взята).
   - `404 Not Found`: Объект не найден.

## Часть 2: API на Express (Node.js)

### Описание реализации
Реализован менеджер задач с хранением данных в JSON-файле.
![Health Check Express](file:///C:/Users/David/.gemini/antigravity/brain/a8ac6be2-836b-43a9-ae75-8a4ecad6cd35/express_health_check_1773947004181.png)
- Использован Express.js как основной фреймворк.
- Middleware для логирования, обработки ошибок и валидации (Joi).
- Файловая система (`fs.promises`) для персистентности данных.
- Безопасность через `helmet` и `express-rate-limit`.

### Ответы на вопросы (Часть 2)
1. **Какие middleware вы использовали и для чего?**
   - `express.json()`: парсинг JSON-тела запроса.
   - `cors`: разрешение кросс-доменных запросов.
   - `helmet`: защита заголовков HTTP.
   - `rateLimit`: защита от DDoS/брутфорса.
   - `errorHandler`: централизованная обработка ошибок.
   - `validation`: проверка входных данных с помощью Joi.

2. **Как работает валидация с Joi в сравнении с Pydantic?**
   Joi — это декларативная библиотека валидации схем, где проверка происходит явно в коде (через middleware). Pydantic в FastAPI интегрирован глубже через типы Python, что позволяет делать валидацию "автоматической" еще до попадания в логику функции.

3. **В чем преимущества файлового хранения данных для этого задания?**
   Простота реализации без необходимости настройки полноценной СУБД (PostgreSQL/MongoDB). Легко для отладки — состояние базы можно посмотреть и поправить прямо в `tasks.json`.

4. **Как бы вы улучшили это API для production использования?**
   - Переход на полноценную БД (например, MongoDB с Mongoose).
   - Добавление аутентификации (JWT).
   - Добавление тестов (Jest/Supertest).
   - Использование Docker для деплоя.
