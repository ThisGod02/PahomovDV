# Система учета сотрудников компании

Комплексная система учета сотрудников компании, реализующая принципы объектно-ориентированного программирования на Python.

## Описание проекта

Проект представляет собой систему управления сотрудниками компании с поддержкой различных типов сотрудников, отделов, проектов и комплексной функциональности для работы с данными.

## Реализованные части лабораторной работы

### Часть 4.1: Инкапсуляция
- Класс `Employee` с приватными атрибутами
- Свойства (property) для доступа к данным
- Валидация входных параметров
- Метод `__str__` для строкового представления

### Часть 4.2: Наследование и абстракция
- Абстрактный класс `AbstractEmployee`
- Классы-наследники: `Manager`, `Developer`, `Salesperson`
- Фабрика сотрудников `EmployeeFactory`
- Переопределение методов расчета зарплат

### Часть 4.3: Полиморфизм и магические методы
- Класс `Department` для управления сотрудниками
- Магические методы для сотрудников (`__eq__`, `__lt__`, `__add__`, `__radd__`)
- Магические методы для отдела (`__len__`, `__getitem__`, `__contains__`, `__iter__`)
- Сериализация/десериализация в JSON
- Компараторы для сортировки

### Часть 4.4: Композиция и агрегация
- Класс `Project` с композицией сотрудников
- Класс `Company` с агрегацией отделов и проектов
- Система валидации и кастомные исключения
- Комплексная сериализация системы
- Экспорт отчетов в CSV
- Бизнес-методы для анализа данных

## Использование

### Запуск примеров

Каждая часть лабораторной работы имеет свой демонстрационный скрипт:

```bash
# Часть 4.1: Инкапсуляция
python examples/demo_part1.py

# Часть 4.2: Наследование и абстракция
python examples/demo_part2.py

# Часть 4.3: Полиморфизм и магические методы
python examples/demo_part3.py

# Часть 4.4: Композиция и агрегация
python examples/demo_part4.py
```

### Пример использования в коде

```python
from src.core.company import Company
from src.core.department import Department
from src.employees.manager import Manager
from src.employees.developer import Developer

# Создание компании
company = Company("TechInnovations")

# Создание отдела
dept = Department("Development")
company.add_department(dept)

# Создание сотрудников
manager = Manager(1, "Алиса", "DEV", 70000, 20000)
developer = Developer(2, "Боб", "DEV", 50000, ["Python"], "senior")

# Добавление в отдел
dept.add_employee(manager)
dept.add_employee(developer)

# Расчет общей зарплаты
total = dept.calculate_total_salary()
print(f"Общая зарплата отдела: {total}")
```

## Основные классы

### Employee
Базовый класс сотрудника с инкапсуляцией данных.

### Manager, Developer, Salesperson
Специализированные классы сотрудников с различной логикой расчета зарплат.

### Department
Класс для управления коллекцией сотрудников с поддержкой полиморфизма.

### Project
Класс проекта с композицией команды сотрудников.

### Company
Класс компании с агрегацией отделов и проектов.

## Особенности реализации

- **Инкапсуляция**: Все атрибуты приватные, доступ через свойства с валидацией
- **Наследование**: Иерархия классов с абстрактным базовым классом
- **Полиморфизм**: Единый интерфейс для работы с разными типами сотрудников
- **Магические методы**: Перегрузка операторов для удобной работы с объектами
- **Композиция и агрегация**: Правильное управление жизненным циклом объектов
- **Сериализация**: Полная поддержка сохранения/загрузки состояния системы
